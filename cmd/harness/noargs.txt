Harness CLI — Quick Reference

Grammar:
  harness <verb> <noun> [id] [flags]              get, update, delete, execute
  harness list <noun> [scope] [flags]             scope narrows results to a parent resource
  harness <verb> <noun>:<qualifier> [id] [flags]  qualified variant (e.g. get pipeline:summary, execute pipeline:input_set)

Auth:
  harness auth status          show resolved account, org, and project
  harness auth login           interactive setup

Discover:
  harness get module <name>          domain model, nouns, and guides for a module
  harness get noun <noun>            fields and commands for a specific noun
  harness <verb> <noun> --help       flags specific to a command

Common verbs:
  list      list resources; optional [scope] narrows results (e.g. pipeline id for executions)
  get       get a single resource by id
  create    create a resource (--file/-f for YAML body, or --set key=value)
  update    update a resource by id (--set key=value, --del key)
  delete    delete a resource by id
  execute   run/trigger a resource by id (e.g. pipeline)

Scope flags (all commands):
  --org, --project                 override resolved scope
  --profile                        use a named auth profile
  --debug                          enable debug logging
  --level project|org|account      set scope for multi-level nouns (e.g. secrets, users)
  --timeout <seconds>              abort after this many seconds; accepts decimals (e.g. 1.5); 0 = no timeout; exits 124

Output flags (list):
  --format table|json|jsonl|csv|tsv|markdown    output format (default: table)
  --json                                        shorthand for --format json
  --columns <ids or exprs>                      customize columns (e.g. "name,org" or "+tags")
  --list-columns                                print available column IDs for this command

Paging flags (list):
  --all                                         fetch all items
  --limit <n>                                   max items to return (default 20)
  --offset <n>                                  start at this item offset (default 0)
  --count                                       print total count instead of results

Output flags (get, create, update):
  --format text|json               output format (default: text)
  --json                           shorthand for --format json
  --fields <ids>                   extract field values, tab-separated (e.g. "name,git_url"); for shell $() capture
  --list-fields                    print available field IDs for this command

Examples:
  harness list project
  harness list pipeline --org my-org --all
  harness get pipeline my-pipeline
  harness execute pipeline my-pipeline
  harness get module pipeline

For detailed help on any command:  harness <verb> <noun> --help
