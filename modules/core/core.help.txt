## Core (core)

The Core module provides authentication, profile management, and CLI self-management
commands that are available regardless of which other modules are installed.

### Authentication

Credentials are stored in named profiles. The default profile is used when no
--profile flag or HARNESS_PROFILE env var is set. Each profile holds an API URL,
account ID, and optional org/project defaults.

  harness auth login              # save credentials to a profile
  harness auth status             # validate the active profile
  harness auth setscope           # set default org/project on a profile
  harness auth profiles           # list all saved profiles
  harness auth logout             # remove a profile

### CLI Management

  harness version                 # print the CLI version
  harness install cli             # install or upgrade the harness binary
  harness install module <name>   # install an external module (e.g. har)
  harness list module             # list all loaded modules
  harness get module <name>       # show nouns and commands for a module
  harness list noun               # list all registered nouns
  harness get noun <noun>         # show fields and commands for a noun
