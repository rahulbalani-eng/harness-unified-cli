## Artifact Registry (har)

Harness Artifact Registry stores and manages software packages across all major
formats. It can act as a proxy for upstream registries (npm, PyPI, Docker Hub)
and enforce firewall policies on package downloads.

### Domain Model

The hierarchy is: registry → artifact → artifact_version → artifact_file.
A registry holds multiple artifacts (packages). Each artifact has one or more
versions. Each version holds one or more files (the actual content). Metadata
can be attached at the registry, artifact, or version level.

Upstream proxying is configured at the registry level. Firewall policy applies
to artifact_version downloads — audit checks a lock file against current policy
without downloading anything.

Metadata (key-value pairs) can be attached at the registry, artifact, or
artifact_version level using the get/update metadata commands. Use --set key=value
to add or update a key and --del key to remove a key.

### Push Variants

Use the explicit variant for the package type you are pushing: `push artifact:maven`,
`push artifact:npm`, `push artifact:python`, etc. Each variant validates the file
format and sets the correct registry type.
